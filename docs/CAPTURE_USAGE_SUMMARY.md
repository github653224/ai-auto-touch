## 🔍 mitmproxy 界面使用

### 查看请求列表

- 每一行代表一个请求
- 显示：方法、URL、状态码、大小、时间

### 查看请求详情

点击任意请求，可以查看：
- **Request**: 请求信息
  - Headers: 请求头
  - Content: 请求体
- **Response**: 响应信息
  - Headers: 响应头
  - Content: 响应体
- **Detail**: 详细信息（时间、大小等）

### 过滤请求

在搜索框输入过滤表达式：

```bash
# 只显示特定域名
~d api.example.com

# 只显示 POST 请求
~m POST

# 只显示错误响应
~c 4  # 4xx 错误
~c 5  # 5xx 错误

# 组合过滤
~d api.example.com & ~m POST
```

### 搜索内容

在搜索框直接输入关键词，可以搜索：
- URL
- 请求头
- 响应头
- 请求体
- 响应体

## ⚠️ 注意事项

### 1. 抓包完成后记得清除代理

```bash
adb shell settings put global http_proxy :0
```

否则手机会一直使用代理，mitmproxy 关闭后无法上网。

### 2. 证书只需安装一次

证书安装后会一直有效，除非：
- 手动删除证书
- 恢复出厂设置
- 重装系统

### 3. 某些应用无法抓包

以下应用通常无法抓包：
- 银行类应用
- 支付类应用（支付宝、微信支付等）
- 证券类应用
- 部分游戏

原因：这些应用使用了证书固定（Certificate Pinning）技术。

### 4. Android 7+ 的限制

Android 7.0 及以上版本，应用默认不信任用户安装的证书。

**影响**：
- 系统浏览器可以抓包
- 部分第三方应用无法抓包

**解决方案**：
- 使用 Root 权限将证书安装到系统目录
- 使用 Xposed 或 Magisk 模块
- 修改应用的网络安全配置（需要重新打包）

### 5. 网络性能影响

使用代理会略微影响网络速度：
- 所有流量需要经过 mitmproxy
- HTTPS 需要解密和重新加密
- 通常延迟增加 10-50ms

## 🔧 故障排除

### 问题 1：手机无法上网

**检查清单**：
```bash
# 1. 检查代理设置
adb shell settings get global http_proxy

# 2. 检查 mitmproxy 是否运行
ps aux | grep mitmweb

# 3. 检查端口
lsof -i :8092

# 4. 测试连通性
ping 192.168.0.109  # 在手机上
```

### 问题 2：只能看到 CONNECT 请求

**原因**：证书未安装或未信任

**解决**：
1. 检查证书：设置 → 安全 → 受信任的凭据 → 用户
2. 重新安装证书
3. 重启手机

### 问题 3：mitmproxy 界面空白

**原因**：没有网络请求

**解决**：
1. 在手机上访问任意网站
2. 确认代理已设置
3. 刷新 mitmproxy 页面

### 问题 4：某些网站无法访问

**原因**：
- 网站使用了 HSTS（强制 HTTPS）
- 证书验证失败

**解决**：
1. 确认证书已正确安装
2. 清除浏览器缓存
3. 尝试访问其他网站
