# 抓包功能使用说明

## 功能概述

本项目集成了完整的网络抓包功能，支持：
- ✅ HTTP/HTTPS 流量抓包
- ✅ 实时查看请求和响应
- ✅ 查看请求头、响应头、请求体、响应体
- ✅ 浏览器流量完全支持
- ✅ 部分 APP 流量支持（需要配合 Frida）

## 快速开始

### 1. 启动抓包服务

抓包服务会在访问抓包页面时自动启动，也可以手动启动：

```bash
# 通过 API 启动
curl -X POST http://localhost:8001/api/v1/mitmproxy/start/EP0110MZ0BC110733W
```

### 2. 设置手机代理

```bash
# 设置代理
bash backend/set_proxy.sh EP0110MZ0BC110733W 192.168.0.109 8091

# 清除代理
bash backend/clear_proxy.sh EP0110MZ0BC110733W
```

### 3. 访问抓包页面

在浏览器中打开：`http://localhost:3002`，进入设备列表，点击设备进入抓包页面。

### 4. 在手机上访问网页

在手机浏览器中访问任何网页，流量会实时显示在抓包页面。

## 功能说明

### 抓包页面功能

1. **流量列表**
   - 显示所有抓到的 HTTP/HTTPS 请求
   - 显示请求方法、URL、状态码、大小、耗时
   - 支持自动刷新（每 2 秒）

2. **流量详情**
   - 点击"详情"按钮查看完整的请求和响应
   - 查看请求头、响应头
   - 查看请求体、响应体（自动格式化 JSON）

3. **操作按钮**
   - **刷新**：手动刷新流量列表
   - **清空**：清空所有抓包记录
   - **自动刷新**：开启/关闭自动刷新

### 代理设置

**代理地址**：显示在抓包页面顶部
- 格式：`192.168.0.109:8091`
- 需要在手机 WiFi 设置中配置此代理

## 证书安装（必须）

要抓取 HTTPS 流量，必须在手机上安装 mitmproxy 证书：

### 安装步骤

1. **设置代理**（见上文）

2. **访问证书下载页面**
   - 在手机浏览器访问：`http://mitm.it`
   - 点击 "Android" 图标下载证书

3. **安装证书**
   - 打开手机 **设置** → **安全** → **加密与凭据**
   - 选择 **从存储设备安装**
   - 选择下载的 `mitmproxy-ca-cert.cer` 文件
   - 给证书命名（如 "mitmproxy"）
   - 选择 **VPN 和应用** 用途

4. **验证**
   - 访问 `https://www.baidu.com`
   - 如果能正常打开，说明证书安装成功

## APP 流量抓包（高级）

### 为什么有些 APP 抓不到？

很多 APP（特别是国内 APP）使用了 **SSL Pinning（证书固定）** 技术：
- 微信、抖音、头条、支付宝等
- 银行类 APP
- 游戏类 APP

这些 APP 会检测代理并拒绝连接。

### 解决方案：Frida + mitmproxy

如果手机已 Root，可以使用 Frida 绕过 SSL Pinning：

#### 1. 安装 Frida Server（一次性）

```bash
# 1. 下载 frida-server
# 访问：https://github.com/frida/frida/releases
# 下载 frida-server-xxx-android-arm64.xz

# 2. 推送到手机
adb push frida-server /data/local/tmp/
adb shell chmod 755 /data/local/tmp/frida-server

# 3. 启动 frida-server（需要 Root）
adb shell su -c "/data/local/tmp/frida-server &"
```

#### 2. 检查 Frida 状态

```bash
# 检查 Frida 是否安装
curl http://localhost:8001/api/v1/frida/check

# 检查设备上的 Frida Server
curl http://localhost:8001/api/v1/frida/device/EP0110MZ0BC110733W/status
```

#### 3. 注入 SSL Unpinning

```bash
# 注入到今日头条
curl -X POST "http://localhost:8001/api/v1/frida/device/EP0110MZ0BC110733W/inject?package_name=com.ss.android.article.news"

# 注入到抖音
curl -X POST "http://localhost:8001/api/v1/frida/device/EP0110MZ0BC110733W/inject?package_name=com.ss.android.ugc.aweme"

# 注入到微信
curl -X POST "http://localhost:8001/api/v1/frida/device/EP0110MZ0BC110733W/inject?package_name=com.tencent.mm"
```

#### 4. 打开 APP 并抓包

注入成功后，打开对应的 APP，流量会被抓取。

## API 接口

### Mitmproxy API

```bash
# 启动抓包服务
POST /api/v1/mitmproxy/start/{device_id}

# 停止抓包服务
POST /api/v1/mitmproxy/stop/{device_id}

# 获取服务状态
GET /api/v1/mitmproxy/device/{device_id}/status

# 获取流量列表
GET /api/v1/mitmproxy/device/{device_id}/flows?limit=100

# 获取流量详情
GET /api/v1/mitmproxy/device/{device_id}/flow/{flow_id}

# 清空流量记录
POST /api/v1/mitmproxy/device/{device_id}/clear
```

### Frida API

```bash
# 检查 Frida 是否安装
GET /api/v1/frida/check

# 检查 Frida Server 状态
GET /api/v1/frida/device/{device_id}/status

# 启动 Frida Server
POST /api/v1/frida/device/{device_id}/start-server

# 注入 SSL Unpinning
POST /api/v1/frida/device/{device_id}/inject?package_name=xxx

# 停止注入
POST /api/v1/frida/stop
```

## 性能优化

### mitmweb 界面优化

为了提升 mitmweb web 界面的性能和稳定性，我们做了以下优化：

1. **限制内容显示行数**
   - 设置 `content_view_lines_cutoff=100`
   - 减少大文件渲染负担

2. **流式传输大文件**
   - 设置 `stream_large_bodies=1m`
   - 超过 1MB 的文件不存储在内存中

3. **降低日志级别**
   - 设置 `termlog_verbosity=warn`
   - 减少不必要的日志输出

4. **禁用调试模式**
   - 设置 `web_debug=false`
   - 提升 web 界面响应速度

### 自定义抓包界面

除了 mitmweb 原生界面，我们还提供了自定义抓包界面：

**优势**：
- 更快的响应速度
- 更简洁的界面
- 自动刷新功能
- 不依赖 mitmweb web 界面

**使用方式**：
- 访问抓包页面，流量会自动显示在自定义列表中
- 点击"打开完整界面"按钮可以访问 mitmweb 原生界面

## 故障排查

### 1. 手机无法上网

**原因**：代理设置错误或证书未安装

**解决**：
```bash
# 清除代理
bash backend/clear_proxy.sh EP0110MZ0BC110733W

# 重新安装证书（见上文）
```

### 2. 抓包页面没有流量

**原因**：
- 代理未设置
- mitmweb 未启动
- 手机未访问网页

**解决**：
```bash
# 检查 mitmweb 状态
curl http://localhost:8001/api/v1/mitmproxy/device/EP0110MZ0BC110733W/status

# 检查代理设置
adb -s EP0110MZ0BC110733W shell "settings get global http_proxy"

# 在手机浏览器访问 http://www.baidu.com
```

### 3. HTTPS 网站无法访问

**原因**：证书未正确安装或未信任

**解决**：
- 重新安装证书
- 确保证书用途选择了 "VPN 和应用"
- 在 **受信任的凭据** 中检查证书是否存在

### 4. APP 显示无网络

**原因**：APP 检测到代理并拒绝连接（正常现象）

**解决**：
- 使用 Frida 绕过 SSL Pinning（需要 Root）
- 或者只抓浏览器流量

### 5. mitmweb 界面卡顿或超时

**原因**：
- 流量过多导致内存占用过高
- 大文件渲染导致浏览器卡顿
- Web 界面资源消耗过大

**解决**：
```bash
# 1. 清空流量记录
curl -X POST http://localhost:8001/api/v1/mitmproxy/device/EP0110MZ0BC110733W/clear

# 2. 重启 mitmweb 服务
curl -X POST http://localhost:8001/api/v1/mitmproxy/stop/EP0110MZ0BC110733W
curl -X POST http://localhost:8001/api/v1/mitmproxy/start/EP0110MZ0BC110733W

# 3. 使用自定义抓包界面（推荐）
# 访问抓包页面，使用自定义列表查看流量，性能更好
```

**建议**：
- 优先使用自定义抓包界面（性能更好）
- 定期清空流量记录
- 避免抓取大文件（视频、下载等）

## 技术架构

```
手机 APP/浏览器
    ↓
手机代理设置 (192.168.0.109:8091)
    ↓
mitmweb (电脑上运行)
    ↓
mitmweb REST API (http://localhost:8191)
    ↓
后端 FastAPI (http://localhost:8001)
    ↓
前端 React (http://localhost:3002)
```

## 注意事项

1. **隐私保护**
   - 抓包会记录所有流量，包括敏感信息
   - 不要在公共网络使用
   - 使用完毕后清除代理

2. **性能影响**
   - 代理会增加网络延迟
   - 建议只在需要时开启

3. **证书安全**
   - mitmproxy 证书仅用于开发测试
   - 不要在生产环境使用

4. **法律合规**
   - 仅抓取自己设备的流量
   - 不要用于非法目的
