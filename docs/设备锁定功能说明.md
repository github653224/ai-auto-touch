## 设备锁定功能说明

### 功能概述

设备锁定功能用于防止多用户同时操作同一台设备，确保设备的独占访问。

**核心特性**：
- ✅ 自动锁定：用户进入设备控制页面时自动获取锁
- ✅ 心跳保活：定期发送心跳，证明用户还在使用（默认 15 秒）
- ✅ 自动释放：用户离开页面或心跳超时（30 秒）自动释放锁
- ✅ 锁定提示：其他用户尝试访问时显示"设备被占用"弹窗
- ✅ 强制获取：管理员可以强制获取锁（抢占）

### 后端 API

#### 1. 获取设备锁

```bash
POST /api/v1/device-lock/device/{device_id}/acquire

# 请求体
{
  "user_id": "user123",
  "user_name": "张三",
  "force": false  # 是否强制获取（管理员权限）
}

# 成功响应
{
  "success": true,
  "session_id": "uuid-xxx-xxx",
  "message": "设备锁获取成功"
}

# 失败响应（设备被占用）
{
  "success": false,
  "message": "设备正在被 李四 使用",
  "lock_info": {
    "session_id": "uuid-yyy-yyy",
    "user_id": "user456",
    "user_name": "李四",
    "locked_at": "2026-01-10T19:30:00",
    "last_heartbeat": "2026-01-10T19:35:00",
    "locked_duration": 300  # 秒
  }
}
```

#### 2. 释放设备锁

```bash
POST /api/v1/device-lock/device/{device_id}/release

# 请求体
{
  "session_id": "uuid-xxx-xxx"
}

# 响应
{
  "success": true,
  "message": "锁已释放"
}
```

#### 3. 发送心跳

```bash
POST /api/v1/device-lock/device/{device_id}/heartbeat

# 请求体
{
  "session_id": "uuid-xxx-xxx"
}

# 成功响应
{
  "success": true,
  "message": "心跳已更新"
}

# 失败响应（锁已丢失）
{
  "success": false,
  "message": "会话 ID 不匹配",
  "should_disconnect": true  # 提示前端断开连接
}
```

#### 4. 查询设备锁状态

```bash
GET /api/v1/device-lock/device/{device_id}/status

# 响应（已锁定）
{
  "locked": true,
  "lock_info": {
    "session_id": "uuid-xxx-xxx",
    "user_id": "user123",
    "user_name": "张三",
    "locked_at": "2026-01-10T19:30:00",
    "last_heartbeat": "2026-01-10T19:35:00",
    "locked_duration": 300
  }
}

# 响应（未锁定）
{
  "locked": false
}
```

#### 5. 列出所有设备锁

```bash
GET /api/v1/device-lock/locks

# 响应
{
  "locks": {
    "EP0110MZ0BC110733W": {
      "session_id": "uuid-xxx-xxx",
      "user_id": "user123",
      "user_name": "张三",
      "locked_at": "2026-01-10T19:30:00",
      "last_heartbeat": "2026-01-10T19:35:00",
      "locked_duration": 300
    }
  }
}
```

#### 6. 强制释放设备锁（管理员）

```bash
POST /api/v1/device-lock/device/{device_id}/force-release

# 响应
{
  "success": true,
  "message": "锁已释放"
}
```

### 前端使用

#### 1. 使用 useDeviceLock Hook

```typescript
import { useDeviceLock } from '../hooks/useDeviceLock';
import DeviceLockModal from '../components/DeviceLockModal';

const MyComponent = () => {
  const [showLockModal, setShowLockModal] = useState(false);
  
  const {
    isLocked,
    sessionId,
    lockInfo,
    acquireLock,
    releaseLock,
    forceAcquireLock,
  } = useDeviceLock({
    deviceId: 'EP0110MZ0BC110733W',
    userId: 'user123',
    userName: '张三',
    enabled: true,
    heartbeatInterval: 15000, // 15 秒
    onLockAcquired: () => {
      console.log('成功获取设备锁');
    },
    onLockLost: () => {
      console.log('设备锁已丢失');
      message.warning('设备锁已丢失，可能被其他用户占用');
    },
    onLockFailed: (lockInfo) => {
      console.log('获取设备锁失败:', lockInfo);
      setShowLockModal(true);
    },
  });

  return (
    <div>
      {isLocked ? (
        <div>
          <p>设备已锁定，会话 ID: {sessionId}</p>
          {/* 你的设备控制界面 */}
        </div>
      ) : (
        <div>
          <p>正在获取设备锁...</p>
        </div>
      )}

      {/* 设备被占用提示弹窗 */}
      <DeviceLockModal
        visible={showLockModal}
        deviceId="EP0110MZ0BC110733W"
        lockInfo={lockInfo}
        onCancel={() => setShowLockModal(false)}
        onForceAcquire={async () => {
          const success = await forceAcquireLock();
          if (success) {
            setShowLockModal(false);
            message.success('成功强制获取设备锁');
          }
        }}
        showForceButton={true} // 管理员才显示
      />
    </div>
  );
};
```

#### 2. Hook 参数说明

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `deviceId` | `string \| null` | 是 | - | 设备 ID |
| `userId` | `string` | 是 | - | 用户 ID |
| `userName` | `string` | 是 | - | 用户名称 |
| `enabled` | `boolean` | 否 | `true` | 是否启用锁定功能 |
| `heartbeatInterval` | `number` | 否 | `15000` | 心跳间隔（毫秒） |
| `onLockAcquired` | `() => void` | 否 | - | 成功获取锁时的回调 |
| `onLockLost` | `() => void` | 否 | - | 锁丢失时的回调 |
| `onLockFailed` | `(lockInfo) => void` | 否 | - | 获取锁失败时的回调 |

#### 3. Hook 返回值

| 字段 | 类型 | 说明 |
|------|------|------|
| `isLocked` | `boolean` | 是否已获取锁 |
| `sessionId` | `string \| null` | 会话 ID |
| `lockInfo` | `LockInfo \| null` | 当前锁信息（失败时） |
| `acquireLock` | `() => Promise<boolean>` | 手动获取锁 |
| `releaseLock` | `() => Promise<void>` | 手动释放锁 |
| `forceAcquireLock` | `() => Promise<boolean>` | 强制获取锁 |

### 工作流程

```
用户 A 进入设备控制页面
    ↓
自动调用 acquireLock()
    ↓
成功获取锁，返回 session_id
    ↓
启动心跳定时器（每 15 秒）
    ↓
用户 A 正常使用设备
    ↓
用户 B 尝试进入同一设备
    ↓
acquireLock() 失败，返回 lock_info
    ↓
显示"设备被占用"弹窗
    ↓
用户 B 可以选择：
    - 返回（等待）
    - 强制获取（管理员）
    ↓
用户 A 离开页面
    ↓
自动调用 releaseLock()
    ↓
锁被释放，其他用户可以获取
```

### 心跳机制

**为什么需要心跳？**
- 防止用户异常退出（浏览器崩溃、网络断开）导致锁无法释放
- 确保锁的持有者仍然活跃

**心跳流程**：
1. 前端每 15 秒发送一次心跳请求
2. 后端更新 `last_heartbeat` 时间
3. 如果超过 30 秒未收到心跳，后端自动释放锁
4. 前端收到心跳失败响应时，显示"锁已丢失"提示

### 配置参数

#### 后端配置

在 `backend/app/services/device_lock_service.py` 中：

```python
# 心跳超时时间（秒）
DeviceLockService(heartbeat_timeout=30)
```

#### 前端配置

在使用 `useDeviceLock` 时：

```typescript
useDeviceLock({
  // ...
  heartbeatInterval: 15000, // 心跳间隔（毫秒）
})
```

**建议配置**：
- `heartbeatInterval` = 15 秒（前端）
- `heartbeat_timeout` = 30 秒（后端）
- 确保 `heartbeat_timeout` > `heartbeatInterval * 2`

### 注意事项

1. **用户标识**
   - 需要实现用户认证系统，获取真实的 `user_id` 和 `user_name`
   - 当前示例使用硬编码，生产环境需要从登录会话获取

2. **心跳间隔**
   - 不要设置太短（增加服务器负担）
   - 不要设置太长（锁释放不及时）
   - 建议：15-30 秒

3. **强制获取**
   - 仅管理员可用
   - 需要实现权限检查
   - 会中断其他用户的操作

4. **页面刷新**
   - 页面刷新会丢失 `session_id`
   - 需要重新获取锁
   - 可以考虑将 `session_id` 存储到 `sessionStorage`

5. **多标签页**
   - 同一用户在多个标签页打开同一设备
   - 会创建多个会话
   - 建议：检测并提示用户
