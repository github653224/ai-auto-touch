🧪 mitmweb 反向代理 iframe 嵌入测试

测试说明

目的: 对比直接访问和反向代理访问的区别

mitmproxy 版本: 10.4.2

设备 ID: test-device

❌ 方式一：直接访问 mitmweb

http://localhost:8191
状态: 正在加载...

⚠️ 预期结果：被 X-Frame-Options 阻止

✅ 方式二：通过反向代理访问

http://localhost:8000/api/v1/mitmproxy/proxy/test-device/
状态: 正在加载...

✅ 预期结果：成功加载（已移除 X-Frame-Options）

📋 测试结果说明

  • 左侧（直接访问）: 浏览器会阻止加载，因为 mitmweb 返回了 X-Frame-Options: DENY 头
  • 右侧（反向代理）: FastAPI 反向代理移除了安全限制头，iframe 可以正常加载
  • 解决方案: 在生产环境中使用反向代理方式，确保安全性和兼容性

🔧 技术实现

  • FastAPI 反向代理移除 X-Frame-Options、Content-Security-Policy 等安全头
  • 添加 CORS 头支持跨域访问
  • mitmweb 只监听 127.0.0.1，通过反向代理对外提供服务
  • 支持 HTTP 和 WebSocket 双向代理
